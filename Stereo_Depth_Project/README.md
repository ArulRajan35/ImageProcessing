# Stereo Vision Depth Estimation using Camera Calibration with MiDaS v3.1 + Custom Fine-Tuning + Streamlit

Production-style beginner-to-intermediate computer vision project for practical monocular depth estimation on road scenes.

## Project Overview

This project builds a complete pipeline:

1. Downloads 1000+ royalty-free road/driving images from Pixabay.
2. Preprocesses data (resize, normalization, optional augmentation, calibration-style undistortion).
3. Uses pretrained MiDaS v3.1 for robust monocular depth inference.
4. Trains a lightweight depth refinement model using MiDaS pseudo-labels (transfer learning workflow).
5. Evaluates model outputs using RMSE, MAE, and relative error (against pseudo-labels unless true depth labels are provided).
6. Runs inference for single images or folders.
7. Provides a Streamlit web app for interactive depth-map generation and download.

> Note: MiDaS is a pretrained model and is not trained from scratch here. The "fine-tuning" stage is implemented as a refinement model trained on pseudo-depth targets generated by MiDaS, which is realistic for datasets without LiDAR depth ground truth.

## Real example: how to read a road-scene depth map

This project estimates **relative depth per pixel**, not object class names. For a typical input:

| What is in the photo | What you usually see in the depth map |
|----------------------|----------------------------------------|
| **Nearby car** | Reads as **closer** than distant structures (compare colors on the car vs the building). |
| **Distant building** | Reads as **farther** than the car. |
| **Road ahead** | Often a **smooth, gradual** change along the lane toward the horizon (near camera at bottom of image in many dashcam shots). |

The Streamlit app includes the same intuition, plus a **vertical depth profile** and **band statistics** so you can see the “gradual road” effect on your own images.

## Installation

1. Create and activate a Python virtual environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

Optional augmentation package (install only if it works in your Python version):

```bash
pip install albumentations
```

## API Key Setup (Pixabay)

1. Create a free Pixabay account and get your API key: [https://pixabay.com/api/docs/](https://pixabay.com/api/docs/)
2. Set environment variable:

### Windows PowerShell

```powershell
$env:PIXABAY_API_KEY="YOUR_API_KEY"
```

### Linux / macOS

```bash
export PIXABAY_API_KEY="YOUR_API_KEY"
```

You can also pass the key directly using `--api-key`.

## Run Pipeline

From project root:

```bash
python scripts/download_dataset.py --target-count 1000
python scripts/preprocess.py --augment
python scripts/train.py
python scripts/evaluate.py
python scripts/inference.py --input dataset/test --use-refiner
streamlit run app/streamlit_app.py
```

## Command Details

### 1) Download dataset

```bash
python scripts/download_dataset.py --target-count 1200 --per-page 200 --max-pages 25
```

- Queries include: road, highway, street, driving, urban road, car dashboard.
- Includes pagination, retry logic, duplicate filtering, and progress bar.

### 2) Preprocess

```bash
python scripts/preprocess.py --image-size 384 --augment
```

- Performs undistortion sample step using a camera calibration model.
- Resizes and normalizes images.
- Splits dataset into train/val/test (70/15/15).

### 3) Train (pseudo-label transfer learning)

```bash
python scripts/train.py --epochs 15 --batch-size 4 --model-key midas_v31_hybrid
```

- Uses MiDaS as teacher to generate pseudo-depth labels.
- Trains a lightweight U-Net style refiner.
- Saves checkpoints, best model, and training logs.
- Uses validation monitoring + early stopping.

### 4) Evaluate

```bash
python scripts/evaluate.py
```

- Computes RMSE, MAE, Relative Error.
- Saves visual comparisons and JSON metrics file.

### 5) Inference

```bash
python scripts/inference.py --input path/to/image.jpg --use-refiner
python scripts/inference.py --input dataset/test --colormap viridis
```

- Supports single image or batch folder inference.
- Saves grayscale depth, colorized depth, and side-by-side comparison.

### 6) Streamlit Web App

```bash
streamlit run app/streamlit_app.py
```

Features:
- Upload image or webcam capture
- MiDaS model selection
- Colormap control
- Confidence threshold slider
- Optional refinement model
- Download depth result

## Engineering Notes

- Robust exception handling and safe file I/O are included across scripts.
- This project focuses on practical deployment quality for demo/academic use, not unrealistic "perfect" depth accuracy.
- Without true ground-truth depth (e.g., KITTI/LiDAR), metrics are relative to pseudo-labels and should be interpreted accordingly.

## Troubleshooting

- **No images downloaded**: Verify `PIXABAY_API_KEY`, internet connection, API quota.
- **Torch Hub MiDaS download issues**: Retry with stable internet; first run downloads weights.
- **`ModuleNotFoundError: No module named 'timm'`**: MiDaS DPT models need PyTorch Image Models. Run `pip install timm` or reinstall: `pip install -r requirements.txt`.
- **CUDA memory errors**: Lower `--batch-size` or switch to CPU.
- **Albumentations install/build error on Windows/Python 3.14**: keep it optional and run preprocessing without `--augment`, or use Python 3.10-3.12 where wheels are more available.
- **Streamlit app fails to load refiner**: Train first or disable "Use trained refinement model".
- **`streamlit` command not found**: run `python -m streamlit run app/streamlit_app.py` to ensure it uses your active environment.

## Academic Explanation (Short)

MiDaS learns monocular depth priors from large multi-dataset training and predicts relative depth.  
For domain adaptation, this project uses pseudo-depth from MiDaS as weak supervision to train a refinement network on road scenes.  
This improves consistency for the target domain while keeping training practical and feasible on standard hardware.
